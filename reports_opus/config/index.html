Task

The data/optimize/config/ datasets are lookup tables of measured software configurations. Unlike Rounds 1‚Äì2 (which tuned a decision tree's hyperparameters and scored R¬≤), this round does direct configuration optimization: a method reveals up to N configurations' performance and returns the best it found, scored by distance-to-heaven (d2h) ‚Äî the Euclidean distance to the ideal point in normalized objective space (lower is better). This is the canonical moot/SE-config-tuning task that TPE is designed for.

Why the method set changed from Round 2. Because evaluating a config's d2h is the legitimate objective (there is no held-out test set to "peek" at), the fair/peek split collapses; and because each config has a single measured value with no cheap/expensive fidelity, BOHB's Hyperband has nothing to halve on. So the 9-method Round-2 set reduces to five: Random, TPE (SOTA optimizer), Greedy surrogate (ablation: no smoothness), INGS (smoothness-aware), and SMAS (smoothness-adaptive). SMAS's Œ≤ is reinterpreted as the smoothness of the d2h response surface over evaluated configs, replacing the trained-learner Œ≤ of Rounds 1‚Äì2.

Experiment

Exp 9 ‚Äî Smoothness-guided configuration optimization (8 datasets √ó 20 repeats, budgets 10/20/30/50, with per-dataset convergence curves).

Headline. SMAS beats TPE at N=30 (5W/1T/2L), beats Random (6W/1T/1L), and beats the no-smoothness Greedy ablation (4W/2T/2L) ‚Äî with the margin over Greedy growing with budget (1W/3T/4L at N=10 ‚Üí 5W/2T/1L at N=50). Mean d2h at N=30: SMAS 0.051, INGS 0.056, TPE 0.095, Greedy 0.074, Random 0.081. This is the strongest result across all three rounds: unlike the regression task where TPE dominated, the smoothness prior makes SMAS the best optimizer here, and TPE only wins clearly on the 39-dim SQL space where the RBF surrogate struggles.
