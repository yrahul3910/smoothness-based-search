Smoothness-Based HPO — Round 2

Follow-up to Round 1 (Exps 1–5). Two new experiments addressing two distinct gaps left by the prior round.

Background

Round 1 extended SMOOTHIE (Yedida & Menzies, 2025) to gradient-free tree learners on 10 SE optimization datasets. Two unresolved threads motivated this round:

  1. Apples-to-oranges baselines. The Exp 3/Exp 5 reports compared INGS (no test access during search) against TPE/BOHB that used r2_score(y_test,…) directly as the search objective. The reported "INGS loses 0W/2T/8L vs BOHB" partly reflected that asymmetry, not INGS's actual quality.
  2. Combining signals. INGS uses val-R² alone; SMOOTHIE-Trees uses β alone; neither uses both. Exp 2 found significant β–R² correlations but dataset-dependent sign. Could one method adaptively combine the two signals?

What's new in Round 2

  • Real BOHB — Exp 3/5 used a TPE stand-in (TPESampler with reduced startup) for "BOHB". Round 2 uses actual BOHB from hpbandster with KDE-based model selection and Hyperband-style sub-sampling fidelity.
  • Fair baselines — for each of TPE, BOHB, Random we run two variants: fair (search objective = val R² on a held-out split of x_train) and peeking (search objective = test R²).
  • SMAS — Smoothness-Augmented INGS. Composite acquisition with an adaptive smoothness term, defined below.

SMAS in one display

score(h) = (1 − weff) · z(R̂²(h)) + weff · sign · z(β̂(h)) + κ · minobs ‖h − obs‖ / √d

where R̂² and β̂ are linear RBF interpolants over (config_vec, observed_val_R²) and (config_vec, observed_β), and the smoothness coefficients are set by the pilot Spearman correlation:

ρ = Spearman(β, R²) over pilot,   sign = ρ ≥ 0 ? +1 : −1,   weff = 0.4 · |ρ|.

The adaptive |ρ| weight makes SMAS auto-dampen the β term when there is no correlation signal — in that case SMAS reduces to INGS-Full. When the correlation is strong, β takes over up to 40% of the acquisition weight.

Experiments

# Experiment Research question Report
6 Fair-comparison reframing How much of TPE/BOHB's reported win was test-set peeking? exp6_fair_comparison.html
7 SMAS — Smoothness-Augmented INGS Does combining val-R² and β with an adaptive weight outperform either alone, and reach parity with TPE/BOHB under fair conditions? exp7_smas.html
8 SMAS-Screen — SMOOTHIE-style two-stage budget Does a cheap low-fidelity screen over many configs (then full-train the top few) recover the original SMOOTHIE speed win for trees? exp8_screen.html
Round-2 bottom line. (6) Most of the prior "TPE/BOHB beat INGS" verdict was test-set peeking — it vanishes on large datasets under fair conditions. (7) SMAS reaches parity with real BOHB using only training data, but TPE remains the quality leader. (8) Porting SMOOTHIE's cheap-screen budget to trees gives only a modest ~1.6× speedup and costs some quality — the trick that was decisive for neural nets largely evaporates because trees are already cheap and BOHB already sub-samples. Net: SMAS (Exp 7) is the method to use; the screen variant is a speed/quality knob, not a free lunch.

Setup

  • Datasets. All 10 CSVs under data/optimize/process/ (COC1000, NASA93, POM3a/b/c/d, XOMO flight/ground/osp/osp2). Inputs and multi-objective targets handled per src/matplotlib.create_surface_data — minimize / maximize columns are normalized and combined as y = √(Σ obj²).
  • HPO space. Same as Exp 5 (DT-only, 5 categorical hyperparameters).
  • Budget. 30 model trains per method per repeat. INGS/SMAS use 15 configs × 2-fold CV; BOHB uses 3 SH iterations with η=3, budget ∈ [1/9, 1] subsample fraction; TPE uses 30 trials with 10 startup; Random uses 30 picks.
  • Repeats. 15 per dataset per method.
  • Stat test. Mann-Whitney U (greater), α=0.05. Tie = neither passes but medians match.

Key files

  • src/methods.py — all 9 method implementations (incl. smas_screen).
  • src/smoothness_trees.py — β computation reused from Round 1.
  • exp_opus.py — driver that produces all three reports + raw JSON.
  • reports_opus/process/exp_opus_raw.json — per-repeat [R², MSE, wall-clock] for full reproducibility.

Reproducing

uv run python exp_opus.py                # full: 10 datasets × 15 repeats × 8 methods (~3.3 hr)
uv run python exp_opus.py --add-screen   # run smas_screen, merge into JSON, build Exp 8
uv run python exp_opus.py --report-only  # rebuild all HTML from cached JSON
uv run python exp_opus.py --prototype    # 3 datasets × 3 repeats (quick sanity)
