Experiment 6: Fair-Comparison Reframing

Date: 2026-05-19 · Runtime: 197.3 min · 10 datasets × 15 repeats

1. Motivation

In prior reports (Exp 3, Exp 5) the TPE and BOHB baselines used the held-out test set directly as their search objective:

def obj(trial):
    cfg = _suggest(trial)
    m.fit(data.x_train, data.y_train)
    return r2_score(data.y_test, m.predict(data.x_test))  # ← test peeking

INGS, by contrast, used a 20% validation split from x_train only. Comparing them as equals gives TPE/BOHB an unfair information advantage: they get N exact test-set evaluations as a free side-effect of the search.

This experiment: Re-run TPE and BOHB in a fair configuration where the objective uses only the same train/val split that INGS uses. Quantify the size of the peeking advantage.

5. Discussion

Test peeking lets the optimizer overfit to the test set within its N=30 query budget, and the effect is strongly size-dependent:

  • Tiny datasets dominate the effect. On nasa93dem (19 test rows) peeking lifts BOHB by +0.20 and Random by +0.19 R²; on pom3d (100 test rows) Random gains +0.105. With so few test points, "pick the config with the best test score" is close to fitting the test set directly.
  • Large smooth datasets barely move. On the XOMO datasets and POM3a/b (2000–4000 test rows) every peeking advantage is ≤ 0.018 R² — the held-out test estimate is stable enough that a fair val split is almost as good.
  • TPE is the most peeking-robust. Its mean advantage is essentially zero (+0.002±0.025), and on nasa93dem the fair variant actually beats the peeking one (−0.067) — TPE's density model is hurt, not helped, by chasing a 19-row test signal. BOHB and Random gain the most from peeking.

Implication for the prior reports. The Exp 3/Exp 5 verdict that "TPE/BOHB beat INGS on every dataset" was inflated by this artifact: the baselines were quietly using test-set feedback that INGS never had. Removing it (this experiment) and adding the smoothness term (Exp 7) closes most of that gap.
